# Annotation Guidelines

## Overview

This dataset contains annotated user utterances for training a conversational AI system.

Each entry includes:

- **user_utterance** – the original user input  
- **intent** – the user's goal  
- **entity_type** – category of extracted information  
- **entity_value** – exact phrase from the user input  

---

## Intent Definitions

- **tech_support** – user reports a technical issue or asks for help with a device  
- **travel_booking** – user wants to book travel (flight, hotel, etc.)  
- **travel_query** – user asks about travel options or availability  
- **banking_request** – user wants to perform a banking-related action  
- **weather_query** – user asks about weather conditions  
- **emotional_support** – user expresses feelings or emotional state  
- **task_management** – user requests scheduling, reminders, or actions  
- **restaurant_query** – user searches for or asks about restaurants  
- **shopping_query** – user asks about products, orders, or shopping  
- **entertainment_request** – user asks for entertainment (movies, jokes, etc.)  
- **general_information** – user asks factual or general knowledge questions  
- **health_information** – user asks about health or wellbeing  
- **casual_conversation** – general or social interaction  

---

## Entity Annotation Rules

- Entities represent important pieces of information needed to fulfill the user’s request  
- Each entity is labeled with:
  - **entity_type** (category)
  - **entity_value** (exact phrase from the user input)

---

## Entity Types Used

- location  
- date  
- time  
- amount  
- device  
- device_issue  
- emotion  
- product  
- cuisine  
- topic  
- action  
- account_type  
- person  

---

## Annotation Principles

1. **Literal extraction**  
   Entities reflect exactly what the user said (no interpretation)

2. **Consistency**  
   The same type of information uses the same entity_type across all rows

3. **Multiple entities**  
   If a sentence contains multiple entities, it is duplicated across rows

4. **Natural language values**  
   Entity values are kept in natural language (e.g., "7 a.m." instead of "7_am")

---

## Notes

- Some utterances may not contain entities  
- Ambiguous inputs are labeled based on the most likely user intent
